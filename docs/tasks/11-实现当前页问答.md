# 任务 11：实现当前页问答

## 1. 目标

本任务的目标是在每一页 slides 下方添加一个问答框，让用户可以针对当前页内容向 LLM 老师提问。

完成后，每一页都有独立的问答历史，用户可以围绕当前页进行追问。

## 2. 实现内容

- 在阅读界面底部添加问题输入框。
- 用户提交问题后，前端调用后端问答接口。
- 后端新增 `POST /api/pages/{page_id}/chat` 接口。
- 后端把当前页上下文发送给 LLM。
- LLM 返回老师回答。
- 保存用户问题和 LLM 回答。
- 切换页面时，只显示当前页的问答历史。

## 3. 实现途径 / 方法 / 手段

- 创建 `ChatMessage` 数据表。
  - `ChatMessage` 表示一条聊天消息。
- `ChatMessage` 至少保存：
  - 所属页面 ID。
  - 消息角色。
  - 消息内容。
  - 创建时间。
- 消息角色建议使用：
  - `user`：用户提出的问题。
  - `assistant`：LLM 老师的回答。
- 用户提交问题后，后端先保存用户消息。
- 后端构造 LLM 输入，建议包含：
  - 课程简介。
  - 当前页文字。
  - 当前页讲稿。
  - 当前页历史问答。
  - 用户最新问题。
- LLM 回答后，后端保存 `assistant` 消息。
- 前端提交问题时需要显示加载状态。
  - 加载状态指系统正在等待 LLM 回答，用户能看到请求还没有结束。
- 如果 LLM 调用失败，应保存或显示失败信息，但不能丢失用户问题。
- 第一版问答只针对当前页，不做跨页全局聊天。

## 4. 合格标准

- 每页底部都有问题输入框。
- 用户输入问题并提交后，能看到 LLM 老师回答。
- 数据库中保存了用户问题和老师回答。
- 刷新页面后，当前页问答历史仍然存在。
- 切换页面后，只显示新页面自己的问答历史。
- LLM 回答明显围绕当前页内容。
- LLM 调用失败时，用户能看到错误提示，页面不会卡死。

## 5. 不合格情况

- 问答历史只保存在前端内存中，刷新后丢失。
- 所有页面共用同一份问答历史。
- 后端没有保存用户问题或 LLM 回答。
- 回答没有使用当前页上下文，导致内容和页面无关。
- LLM 请求失败后输入框一直处于加载状态。
- 当前任务加入全局聊天、多用户聊天等复杂功能，导致当前页问答无法单独验收。
