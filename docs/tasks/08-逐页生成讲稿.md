# 任务 08：逐页生成讲稿

## 1. 目标

本任务的目标是让 LLM 为 PDF slides 的每一页生成老师讲课用的文字讲稿。

完成后，用户在阅读任意一页 slides 时，都能看到与当前页内容对应的讲解文字。

## 2. 实现内容

- 为每一页调用一次 LLM。
- 每页 LLM 输入包括：
  - 课程简介。
  - 当前页文字。
  - 当前页截图。
  - 当前页页码。
- LLM 输出当前页讲课重点。
- 将输出保存到 `Page.lecture_notes`。
- 更新每页生成状态。
- 某一页生成失败时，不影响其他页面继续生成。
- 前端能看到每页讲稿是否已生成。

## 3. 实现途径 / 方法 / 手段

- 在课程简介生成完成后，启动逐页讲稿生成流程。
- 使用后台任务逐页处理。
  - 后台任务指不阻塞用户页面操作、在服务器内部继续运行的任务。
  - 这样用户不需要一直等待所有页面生成完成。
- 每页调用 `LLMClient` 的图文输入能力。
- 当前页截图用于帮助模型理解图表、布局、公式和视觉信息。
- Prompt 需要要求 LLM 扮演老师，输出适合课堂讲解的内容。
- 输出建议包含：
  - 本页核心观点。
  - 老师讲解词。
  - 需要提醒学生注意的地方。
  - 与前后页面的衔接。
- `Page.status` 建议支持：
  - `pending`：等待生成。
  - `processing`：正在生成。
  - `ready`：生成成功。
  - `failed`：生成失败。
- 如果某页失败，应记录错误原因，方便后续重试。
- 第一版可以顺序生成，不要求并发生成。
  - 顺序生成实现简单，也能降低 API 限流风险。

## 4. 合格标准

- 对于一个 3 页 PDF，系统最终能为 3 页分别生成讲稿。
- 每页讲稿保存到对应 `Page.lecture_notes`。
- 刷新页面后，讲稿仍然存在。
- 某一页生成失败时，其他页面仍可继续生成或显示已有结果。
- 前端能区分页面讲稿的生成中、成功、失败状态。
- 讲稿内容围绕当前页，而不是整份文档的泛泛总结。
- 当前任务不要求实现拖动文字块和页面问答。

## 5. 不合格情况

- 所有页面共用同一段讲稿。
- 讲稿没有保存到数据库。
- 某一页失败导致整个文档都无法查看。
- 前端无法知道讲稿是否还在生成。
- LLM 输入没有包含当前页信息，导致输出和页面内容不对应。
- 当前任务混入拖拽 UI 或问答功能，导致逐页生成无法单独验收。
