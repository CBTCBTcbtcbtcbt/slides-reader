# 任务 01：创建项目骨架

## 1. 目标

本任务的目标是建立一个最小可运行的前后端项目，让后续功能可以在稳定的基础上逐步添加。

完成后，项目应该具备两个最基础的能力：

- 后端可以启动，并提供一个健康检查接口。
- 前端可以启动，并能请求后端接口确认连接成功。

这里的“项目骨架”指的是目录结构、基础依赖、启动方式和最小页面，不包含 PDF 上传、数据库、LLM 调用等业务功能。

## 2. 实现内容

- 创建 `backend/` 目录，用来存放 Python 后端代码。
- 后端使用 `FastAPI`。
  - `FastAPI` 是一个 Python Web 框架，用来编写 HTTP API。
  - HTTP API 可以理解为前端页面和后端程序之间约定好的访问入口。
- 后端提供 `GET /api/health` 接口。
  - 这个接口只负责返回后端当前是否正常运行。
- 创建 `frontend/` 目录，用来存放 Web 前端代码。
- 前端使用 `React + TypeScript + Vite`。
  - `React` 是构建网页界面的 JavaScript 库。
  - `TypeScript` 是带类型检查的 JavaScript，可以减少变量和接口使用错误。
  - `Vite` 是前端开发工具，用来启动开发服务器和打包前端代码。
- 前端首页请求后端的 `/api/health`。
- 前端根据请求结果显示“后端连接成功”或“后端连接失败”。
- 添加基础项目说明，记录后端和前端分别如何启动。

## 3. 实现途径 / 方法 / 手段

- 后端建议创建独立的 Python 虚拟环境。
  - 虚拟环境是一个隔离的 Python 依赖目录，避免不同项目之间的依赖互相影响。
- 后端依赖至少包含：
  - `fastapi`：用于创建 API。
  - `uvicorn`：用于启动 FastAPI 服务。
- 后端主入口建议命名为 `main.py`。
- `/api/health` 返回简单 JSON，例如包含 `status` 字段。
  - JSON 是前后端常用的数据格式，类似 Python 中的字典。
- 前端通过 Vite 创建 React TypeScript 项目。
- 前端请求后端时，先使用固定后端地址，例如 `http://localhost:8000/api/health`。
- 如果前端和后端端口不同，需要在后端允许跨域请求。
  - 跨域请求指的是网页从一个地址访问另一个地址的 API。
  - 开发环境中通常需要配置 `CORS`。
  - `CORS` 是浏览器的安全机制，用来控制网页是否允许访问不同域名或端口的后端。

## 4. 合格标准

- 在本地可以单独启动后端服务。
- 访问 `GET /api/health` 时，后端返回 HTTP 200。
- `/api/health` 返回的数据中能明确看出后端状态正常。
- 在本地可以单独启动前端服务。
- 打开前端页面后，页面能显示后端连接成功。
- 当前任务完成后，不需要上传 PDF，不需要连接数据库，不需要调用 LLM。
- 项目中有清楚的启动说明，能让开发者知道如何启动前端和后端。

## 5. 不合格情况

- 后端无法启动。
- 前端无法启动。
- 前端页面不能请求后端健康检查接口。
- `/api/health` 接口不存在，或访问后返回错误。
- 项目目录混乱，前端和后端代码没有清晰分开。
- 当前任务中混入 PDF 上传、数据库、LLM 调用等后续功能，导致任务边界不清晰。
