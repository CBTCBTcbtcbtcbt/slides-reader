# 任务 13：整理界面布局

## 1. 目标

本任务的目标是把已经实现的上传、阅读、讲稿、拖动文字块和问答功能整理成一个可用的学习界面。

完成后，用户进入文档后，可以顺畅完成看课程简介、阅读 slides、查看讲解、翻页和提问这些核心操作。

## 2. 实现内容

- 设计并整理文档阅读页整体布局。
- 整体布局大致仿照PowerPoint
- 左上角设置“文件”按钮，按下跳转至文件页面
- 文件页面可以上传、删除、重命名、选择文件。
- 文件页面直观的显示当前打开的是哪个文件。
- 文件页面添加设置按钮，按下跳转设置页面，可以修改设置 
- 在阅读页中提供课程简介入口。
- 中间区域展示 PDF 当前页。
- PDF 周边展示可拖动讲课文字块。
- 底部固定显示当前页问答框。
- 页面中明显展示翻页按钮。
- 处理小屏幕下的布局适配。
- 保证主要 UI 元素不会互相遮挡。
- 界面尽可能大，不做两侧宽度的限制等任何限制。

## 3. 实现途径 / 方法 / 手段

- 阅读页可以分成几个明确区域：
  - 顶部区域：文件与其他操作按钮，文档标题、处理状态、课程简介入口。
  - 中间区域：PDF 阅读区和讲课文字块。
  - 底部区域：当前页问答框。
- 课程简介使用独立简介页。
  - 第一版推荐使用可展开区域，避免用户离开阅读页面。
- PDF 阅读区需要有稳定尺寸。
  - 稳定尺寸可以避免文字块、按钮、页码频繁跳动。
- 翻页按钮要明显。
- 问答框建议固定在底部。
  - 用户阅读任意页面时都能快速提问。
- 文字块默认位置应避开 PDF 正中央和底部问答框。
- 小屏幕下可以让简介和讲稿区域改为上下排列。
- 对长讲稿和长问答历史，需要使用滚动区域。
  - 滚动区域可以避免内容过长把整个页面撑坏。
- 本任务以可用性为目标，不追求复杂视觉设计。

## 4. 合格标准

- 用户进入文档阅读页后，可以看到文档标题和当前处理状态。
- 用户能打开或查看课程简介。
- 用户能看到 PDF 当前页。
- 用户能看到当前页讲稿文字块。
- 用户能使用明显按钮翻页。
- 用户能在底部输入问题并查看当前页问答。
- 主要按钮、PDF、文字块、问答框没有严重重叠。
- 在常见桌面浏览器尺寸下，核心功能都可用。
- 在较窄屏幕下，页面仍然能阅读和操作。

## 5. 不合格情况

- 功能虽然存在，但分散在不同页面，用户无法顺畅完成学习流程。
- 翻页按钮不明显。
- 问答框遮挡 PDF 或文字块。
- 文字块遮挡底部输入框且无法移开。
- 页面尺寸变化后出现明显内容重叠。
- 课程简介只能在数据库中看到，前端没有入口。
