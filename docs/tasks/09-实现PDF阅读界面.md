# 任务 09：实现 PDF 阅读界面

## 1. 目标

本任务的目标是在网页中展示用户上传的原始 PDF slides，并提供明显、简便的翻页方式。

完成后，用户可以像阅读课件一样在网页中查看每一页 slides，而不是只能下载 PDF 文件。

## 2. 实现内容

- 前端加载后端保存的原始 PDF 文件。
- 在页面中显示 PDF 当前页。
- 添加当前页码和总页数显示。
- 添加“上一页”和“下一页”按钮。
- 支持鼠标中键翻页。
- 翻页后，当前页码正确更新。
- 当前任务只负责 PDF 阅读，不负责展示 LLM 讲稿文字块。

## 3. 实现途径 / 方法 / 手段

- 后端提供 `GET /api/documents/{document_id}/file` 接口。
  - 该接口返回原始 PDF 文件。
- 前端可以使用 `pdf.js` 或 `react-pdf`。
  - `pdf.js` 是浏览器中渲染 PDF 的常用库。
  - `react-pdf` 是基于 React 封装的 PDF 展示组件。
- 前端进入阅读页时，根据 `document_id` 加载 PDF。
- 前端维护当前页码状态。
  - 状态可以理解为页面当前记住的数据，例如当前正在看第几页。
- 点击“上一页”时，如果当前不是第一页，就切换到前一页。
- 点击“下一页”时，如果当前不是最后一页，就切换到后一页。
- 鼠标中键翻页需要监听浏览器鼠标事件。
  - 第一版可以定义为：在 PDF 阅读区域鼠标中键向下滑动时进入下一页，向上滑动时进入上一页。
  - 如果已经是第一页或最后一页，则保持当前页不变。
- 翻页按钮必须足够明显，不能只依赖鼠标中键。

## 4. 合格标准

- 打开文档阅读页后，能在网页中看到 PDF 页面。
- PDF 内容清晰，不是空白区域。
- 页面显示当前页码和总页数。
- 点击“上一页”和“下一页”按钮可以正确翻页。
- 第一页点击“上一页”不会报错。
- 最后一页点击“下一页”不会报错。
- 在 PDF 阅读区域点击鼠标中键可以触发翻页。
- 当前任务不要求显示讲稿文字块和问答框。

## 5. 不合格情况

- 网页只能下载 PDF，不能直接阅读。
- PDF 显示为空白。
- 翻页后页码和实际页面不一致。
- 第一页或最后一页翻页时出现错误。
- 只支持鼠标中键，不提供按钮翻页。
- 当前任务加入拖动文字块和问答功能，导致阅读界面无法单独验收。
