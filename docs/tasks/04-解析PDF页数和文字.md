# 任务 04：解析 PDF 页数和文字

## 1. 目标

本任务的目标是让后端能够读取上传的 PDF slides，识别总页数，并提取每一页中的文字内容。

完成后，系统就能知道一份 slides 有多少页，也能为后续生成课程简介和逐页讲稿准备文本材料。

## 2. 实现内容

- 引入 `PyMuPDF` 处理 PDF。
  - `PyMuPDF` 是 Python 中常用的 PDF 处理库，可以读取页面、提取文字、渲染图片。
- 在数据库中创建 `Page` 数据表。
- 每个 PDF 页面对应一条 `Page` 记录。
- `Page` 至少保存以下信息：
  - 页面 ID。
  - 所属文档 ID。
  - 页码。
  - 页面提取出的文字。
  - 页面处理状态。
- 上传 PDF 后，后端解析页数。
- 后端逐页提取文字并写入数据库。
- 文档状态根据处理结果更新。

## 3. 实现途径 / 方法 / 手段

- 在上传 PDF 后启动解析流程。
- 使用 `PyMuPDF` 打开已保存的 PDF 文件。
- 读取 PDF 的页面数量。
- 从第一页到最后一页逐页处理。
- 对每一页调用文字提取方法，得到该页文本。
- 如果某一页没有可提取文字，仍然要创建 `Page` 记录。
  - slides 中可能有纯图片页。
  - 没有文字不代表页面不存在。
- 页码建议使用用户习惯的编号，从 1 开始。
- `Page.status` 建议先支持：
  - `pending`：等待后续处理。
  - `ready`：页面文字提取完成。
  - `failed`：页面解析失败。
- 解析完成后，`Document` 可以记录总页数，或者通过查询 `Page` 数量得到总页数。
- 如果 PDF 文件损坏，后端需要把文档状态设置为 `failed`，并记录错误原因。

## 4. 合格标准

- 上传一个多页 PDF 后，数据库中为每一页创建一条 `Page` 记录。
- `Page.page_number` 和 PDF 实际页码一致。
- 对于包含文字的页面，`Page.text` 中能看到提取出的文字。
- 对于没有文字的页面，也存在对应 `Page` 记录。
- 后端能返回文档总页数。
- PDF 损坏或无法打开时，系统不会崩溃，而是把文档标记为失败。
- 当前任务不要求生成页面截图，不要求调用 LLM。

## 5. 不合格情况

- 后端无法识别 PDF 总页数。
- 数据库中缺少某些页面记录。
- 页码从前端角度看和 PDF 实际页码不一致。
- 遇到空文字页面就跳过该页。
- PDF 解析失败导致整个后端服务停止。
- 当前任务直接加入 LLM 讲稿生成，导致 PDF 解析能力无法单独测试。
