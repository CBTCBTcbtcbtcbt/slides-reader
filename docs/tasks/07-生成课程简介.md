# 任务 07：生成课程简介

## 1. 目标

本任务的目标是在上传并解析完整份 PDF 后，让 LLM 先生成一个整体课程简介。

课程简介会放在 WebUI 最前面，帮助学生在阅读每一页 slides 之前了解这堂课的主题、结构和学习重点。

## 2. 实现内容

- 从数据库读取当前文档所有页面的文字。
- 将整份文档内容整理为 LLM 输入。
- 调用 `LLMClient` 生成课程简介。
- 要求 LLM 扮演老师，而不是普通摘要工具。
- 将生成结果保存到 `Document.course_summary`。
- 前端在文档详情或阅读页最前面展示课程简介。
- 课程简介生成中和失败时，前端显示对应状态。

## 3. 实现途径 / 方法 / 手段

- 后端在 PDF 页面文字解析完成后，启动课程简介生成。
- 整理输入时，应包含：
  - 文档标题。
  - 总页数。
  - 每页提取的文字。
- 如果文档很长，需要考虑输入长度限制。
  - 输入长度限制指模型一次请求能接收的最大文本量。
  - 第一版可以先做简单截断或压缩，确保请求不会超出模型限制。
- Prompt 需要明确说明角色。
  - Prompt 是发送给 LLM 的指令文本。
  - 应要求模型以“老师备课”的方式输出课程导读。
- 输出建议包含：
  - 课程主题。
  - 适合的学习对象。
  - 主要知识点。
  - 推荐学习顺序。
  - 学完后应掌握的内容。
- 保存结果后，更新文档状态或简介状态。
- 如果 LLM 调用失败，需要记录错误，不应删除已上传文档和已解析页面。

## 4. 合格标准

- 上传并解析 PDF 后，系统会调用 LLM 生成整体课程简介。
- `Document.course_summary` 中保存了 LLM 返回的简介文本。
- 刷新页面后，课程简介仍然存在。
- 前端可以展示课程简介。
- 简介内容不是简单复制原文，而是面向学生的课程导读。
- LLM 调用失败时，前端能看到失败状态或错误提示。
- 当前任务不要求逐页生成讲稿。

## 5. 不合格情况

- 课程简介只显示在内存中，刷新后丢失。
- 简介没有写入数据库。
- Prompt 没有要求 LLM 扮演老师，导致输出只是普通摘要。
- 文档较长时直接请求失败，且没有错误处理。
- LLM 调用失败导致整个文档记录不可用。
- 当前任务同时实现逐页讲稿和问答，导致简介生成功能无法单独验收。
