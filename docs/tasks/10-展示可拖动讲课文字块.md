# 任务 10：展示可拖动讲课文字块

## 1. 目标

本任务的目标是在 PDF 阅读界面中展示 LLM 老师的讲课文字，并让这些文字块可以拖动。

完成后，用户可以一边看 slides，一边把讲课文字块移动到合适的位置，避免遮挡重点内容。

## 2. 实现内容

- 在每页 PDF 周边显示讲课文字块。
- 文字块内容来自当前页的 `Page.lecture_notes`。
- 使用可拖动组件实现拖拽。
- 拖动结束后，保存文字块的新位置。
- 切换页面时，加载当前页自己的文字块位置。
- 刷新页面后，文字块位置保持不变。
- 第一版每页默认一个讲课文字块。

## 3. 实现途径 / 方法 / 手段

- 前端可以使用 `react-rnd`。
  - `react-rnd` 是 React 中常用的拖动和缩放组件库。
  - 它可以让一个区域被鼠标拖动，也可以调整宽高。
- 后端创建 `NoteBlock` 数据表。
  - `NoteBlock` 表示一个可拖动的文字块。
- `NoteBlock` 至少保存：
  - 所属页面 ID。
  - 文字内容。
  - 横向位置 `x`。
  - 纵向位置 `y`。
  - 宽度 `width`。
  - 高度 `height`。
- 当某页讲稿生成完成后，可以为该页创建默认 `NoteBlock`。
- 前端加载当前页详情时，同时加载文字块信息。
- 用户拖动结束后，前端调用 `PATCH /api/note-blocks/{note_block_id}` 保存位置。
- 坐标建议以阅读区域为参考。
  - 这样不同页面切换时，文字块位置更容易保持稳定。
- 第一版不需要支持创建多个自定义文字块。

## 4. 合格标准

- 当前页讲稿生成后，阅读界面能看到对应文字块。
- 文字块内容与当前页讲稿一致。
- 用户可以用鼠标拖动文字块。
- 拖动结束后刷新页面，文字块位置仍然保持。
- 切换到另一页时，显示另一页自己的文字块。
- 不同页面的文字块位置互不影响。
- 文字块不会遮挡固定的翻页按钮和底部问答区域。
- 当前任务不要求实现页面问答。

## 5. 不合格情况

- 讲稿只能固定显示，不能拖动。
- 拖动后位置没有保存。
- 刷新页面后文字块回到默认位置。
- 所有页面共用同一个文字块位置。
- 文字块遮挡关键操作按钮，导致无法翻页或提问。
- 当前任务加入问答逻辑，导致拖动文字块功能无法单独验收。
