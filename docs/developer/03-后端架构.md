# 后端架构

## 总体说明

后端已经从单文件 `backend/main.py` 拆分为按职责组织的模块。对外 API 路径、响应字段和启动方式保持不变，`launch.ps1` 以及 `uvicorn main:app` 仍然可用。

`backend/main.py` 现在只做兼容入口：

```python
from app import app
```

真实的 FastAPI 应用创建、CORS 配置和路由挂载都在 `backend/app.py` 中完成。

## 当前目录结构

```text
backend/
  main.py                  # 兼容 uvicorn main:app 的入口，只导出 app
  app.py                   # FastAPI 应用创建、CORS 配置和路由挂载
  config.py                # 路径、默认 prompt 和常量
  database.py              # SQLite 初始化和连接
  schemas.py               # Pydantic 请求体
  llm_client.py            # LLM 配置校验、序列化和 OpenAI-compatible 调用
  pdf_service.py           # PDF/PPT 转换、文件安全检查、PDF 解析和页面截图
  generation_service.py    # 课程简介、逐页讲稿和当前页问答生成流程
  repositories/
    documents.py           # documents 表读写
    pages.py               # pages 表读写
    note_blocks.py         # note_blocks 表读写
    chat_messages.py       # chat_messages 表读写
    settings.py            # app_settings 表读写
  routes/
    health.py              # 健康检查接口
    documents.py           # 文档、文件、状态和讲稿生成控制接口
    pages.py               # 当前页问答和讲稿文字块接口
    llm.py                 # LLM 配置和测试接口
  tests/
    test_api_contract.py   # 后端 API 契约测试
```

## 应用入口

`app.py` 提供 `create_app()`：

- 创建 `FastAPI` 应用对象。
- 配置 `CORSMiddleware`，允许本地 Vite 前端访问。
- 挂载 `health`、`documents`、`pages`、`llm` 四组路由。
- 在 `startup` 阶段调用 `init_database()` 初始化 SQLite。

`FastAPI` 是 Python Web 框架。它会根据路由函数和 Pydantic 请求体自动生成 OpenAPI schema。开发时可以访问：

```text
http://127.0.0.1:8000/docs
```

这个页面是 Swagger UI，用来手动调试接口。

## 路径和配置

`config.py` 统一保存后端常量：

```text
BASE_DIR = backend/
DEFAULT_STORAGE_DIR = 项目根目录/storage
STORAGE_DIR = 环境变量 SLIDES_READER_STORAGE_DIR 或 DEFAULT_STORAGE_DIR
DOCUMENT_STORAGE_DIR = STORAGE_DIR/documents
PAGE_IMAGE_STORAGE_DIR = STORAGE_DIR/pages
CONVERSION_TEMP_DIR = STORAGE_DIR/conversion-tmp
DATABASE_PATH = STORAGE_DIR/app.db
```

`SLIDES_READER_STORAGE_DIR` 用于测试或临时运行时切换存储目录，避免污染项目真实 `storage/`。

默认 prompt 也放在 `config.py` 中，包括：

- `DEFAULT_COURSE_SUMMARY_PROMPT`
- `DEFAULT_LECTURE_NOTES_PROMPT`
- `DEFAULT_PAGE_CHAT_PROMPT`

## 数据库层

`database.py` 负责 SQLite 初始化和连接：

- `init_database()`：创建存储目录，创建或补齐 `documents`、`pages`、`note_blocks`、`chat_messages`、`app_settings` 表。
- `get_database_connection()`：返回 `sqlite3.Connection`，并设置 `row_factory = sqlite3.Row`，让查询结果可以按字段名读取。

SQLite 是本地文件数据库。当前后端每次操作都打开一个连接，操作完成后关闭，没有使用连接池。

## Repository 层

`repositories/` 目录把数据库读写从路由和业务流程中拆出来。Repository 可以理解为“数据访问层”，它只关心如何读写数据库，不关心 HTTP 请求。

主要边界：

- `documents.py`：文档列表、文档状态、重命名、删除、课程简介状态、讲稿暂停状态。
- `pages.py`：页面记录、页面截图路径、讲稿状态、页面列表聚合。
- `note_blocks.py`：讲稿文字块创建、同步内容、位置更新和最小宽高校验。
- `chat_messages.py`：当前页问答历史写入和读取。
- `settings.py`：LLM 配置读写。

序列化函数也放在对应 repository 中，例如把数据库里的 `documents.id` 转成 API 响应里的 `document_id`。

## 请求体模型

`schemas.py` 保存 Pydantic 请求体。Pydantic 是 Python 数据校验库，FastAPI 会用它自动校验 JSON 字段。

当前请求体包括：

- `RenameDocumentRequest`：文档重命名。
- `LLMConfigUpdateRequest`：保存 LLM 配置。
- `LLMTestRequest`：测试 LLM 连接。
- `PageChatRequest`：当前页问答。
- `NoteBlockPositionUpdateRequest`：保存讲稿文字块坐标和尺寸。

其中讲稿文字块宽高在后端也会校验，最小宽度是 `120`，最小高度是 `80`。

## PDF 服务

`pdf_service.py` 负责文件和 PDF 相关逻辑：

- 校验上传文件后缀是否为 `.pdf`、`.ppt` 或 `.pptx`。
- PDF 直接保存为系统实际使用文件。
- PPT/PPTX 先保存到临时目录，再通过 LibreOffice 转为 PDF。
- 使用 PyMuPDF 解析 PDF 页面文字。
- 使用 PyMuPDF 渲染页面 PNG 截图。
- 检查数据库中的文件路径是否仍在允许目录内，避免路径越界访问。

上传入口仍然是：

```text
POST /api/documents
```

上传成功后，`documents.file_path` 永远指向系统实际使用的 PDF。后续阅读器、页面解析、截图、删除和 LLM 输入都围绕这个 PDF 工作。

## LLM 客户端

`llm_client.py` 是后端统一的 LLM 调用入口。LLM 是 Large Language Model，也就是“大语言模型”。

主要职责：

- 读取并规范化 LLM 配置。
- 序列化配置响应，API Key 只返回掩码，不返回明文。
- 生成 OpenAI-compatible `/chat/completions` 请求地址。
- 把页面截图编码为 `data:image/png;base64,...`。
- 发起纯文本或图文请求。
- 把常见错误转换成明确 HTTP 错误。

常见错误处理：

- 配置缺失：`400`
- LLM 服务返回 HTTP 错误：`502`
- 网络连接失败：`502`
- 请求超时：`504`
- 响应结构不符合 OpenAI-compatible 格式：`502`
- 空回答：`502`

## 生成服务

`generation_service.py` 保存 AI 生成相关业务流程：

- `generate_course_summary(document_id)`：生成课程简介。
- `generate_document_lecture_notes(document_id)`：按文档逐页生成讲稿。
- `generate_single_page_lecture_notes(document, page)`：重新生成单页讲稿。
- `answer_page_question(page_id, question)`：当前页问答，并保存 user/assistant 消息。

逐页讲稿生成使用进程内锁：

```python
LECTURE_NOTES_GENERATION_LOCKS: dict[str, threading.Lock] = {}
LECTURE_NOTES_GENERATION_LOCKS_GUARD = threading.Lock()
```

这个锁只在单进程内有效。未来如果使用多个 worker 进程，需要改成数据库锁、任务队列锁或其他跨进程机制。

讲稿生成成功后会同步 `note_blocks`：

- 如果页面没有文字块，就创建默认文字块。
- 如果页面已有文字块，只更新内容，不覆盖用户拖动过的位置和尺寸。

## 路由层

`routes/` 只处理 HTTP 层输入输出，复杂逻辑交给 service 或 repository。

当前路由分组：

| 文件 | 主要接口 |
| --- | --- |
| `health.py` | `GET /api/health` |
| `documents.py` | 文档列表、上传、PDF 文件、页面截图、状态、重命名、删除、课程简介和逐页讲稿生成控制 |
| `pages.py` | `POST /api/pages/{page_id}/chat`、`PATCH /api/note-blocks/{note_block_id}` |
| `llm.py` | `GET/PATCH /api/llm/config`、`POST /api/llm/test` |

接口路径和响应字段保持原契约不变。重构时如果发现字段变化，应视为回归并修正。

## 后台任务

后端仍使用 FastAPI 的 `BackgroundTasks` 提交生成任务。`BackgroundTasks` 会在响应返回后继续在同一个服务进程里执行函数。

限制：

- 适合本地开发和轻量使用。
- 如果后端进程退出，后台任务会中断。
- 未来如果需要任务恢复、队列、并发控制或多用户隔离，应引入专门任务系统。

## 测试保护

后端已经新增 `backend/tests/test_api_contract.py`，用于保护重构后的 API 契约。测试使用临时 `SLIDES_READER_STORAGE_DIR`，不会污染项目 `storage/`。

运行命令：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest
```
