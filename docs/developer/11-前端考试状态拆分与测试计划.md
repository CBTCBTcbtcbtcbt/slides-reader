# 前端考试状态拆分与测试计划

本文档记录本轮前端优化的执行边界。目标是减少 `App.tsx` 对考试、错题本、阶段考试状态的直接管理，并为前端补上 `Vitest` 与 `Testing Library` 自动化测试。

## 背景说明

`App.tsx` 仍然承担了过多职责。它既负责路由、文档阅读器、LLM 设置，也直接维护考试列表、答题页、结果页、错题本和阶段考试的状态。

`hook` 是 React 中复用状态逻辑的函数。把一组相关状态和操作放入 hook 后，组件只负责渲染和传递事件，业务状态会更集中，也更容易写测试。

`Vitest` 是适配 `Vite` 项目的测试框架。`Testing Library` 是 React 组件测试工具，重点验证用户能看到什么、点击什么，而不是验证组件内部实现细节。

## 本轮目标

- 增加前端测试脚本和测试环境配置。
- 新增一个考试工作流 hook：`frontend/src/hooks/useExamWorkflows.ts`。
- 把普通试卷、错题本、阶段考试相关状态和 API 操作迁入该 hook。
- 让 `App.tsx` 保留路由、文档和阅读器职责，只消费 hook 返回的状态与操作。
- 增加最小但有效的测试，覆盖异步考试状态判断和阶段考试创建页关键交互。

## 非目标

- 不重写前端路由结构。
- 不改后端 API 契约。
- 不重构阅读器、LLM 设置、文档上传、PDF 交互等无关模块。
- 不引入大型状态管理库，例如 `Redux` 或 `Zustand`。

## 文件变更计划

- 修改 `frontend/package.json`：增加 `test`、`test:watch` 脚本，并加入测试依赖。
- 修改 `frontend/vite.config.ts`：增加 `Vitest` 的 `jsdom` 环境和 setup file。
- 新增 `frontend/src/test/setup.ts`：加载 `@testing-library/jest-dom/vitest`。
- 新增 `frontend/src/hooks/useExamWorkflows.ts`：集中考试、错题本、阶段考试状态和操作。
- 新增 `frontend/src/hooks/useExamWorkflows.test.ts`：先验证 `isAsyncExamStatus`，后续可继续补 hook 行为测试。
- 新增 `frontend/src/components/PhaseExamCreateView/PhaseExamCreateView.test.tsx`：验证创建阶段考试时会提交选择的文档、名称和难度。
- 修改 `frontend/src/App.tsx`：删除迁出的考试状态和函数，改为调用 `useExamWorkflows`。
- 更新 `docs/developer/README.md`：把本文档加入阅读列表。
