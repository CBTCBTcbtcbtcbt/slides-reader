# 前端架构

## 总体说明

前端当前集中在 `frontend/src/App.tsx` 和 `frontend/src/styles.css`。

`App.tsx` 是主要 React 组件，包含：

- 类型定义。
- 页面级状态。
- 后端 API 调用。
- 文档管理逻辑。
- LLM 配置逻辑。
- PDF 阅读器逻辑。
- 讲稿文字块拖动和缩放逻辑。
- 当前页问答逻辑。
- JSX 页面渲染。

`styles.css` 包含全局样式、文件页、设置页、阅读器、侧栏、文字块和响应式布局。

这是一种第一版单组件结构。后续开发建议拆分，但拆分前应先补测试。

## 入口文件

`frontend/src/main.tsx` 负责把 `App` 挂载到页面：

```tsx
createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <App />
  </StrictMode>,
);
```

`StrictMode` 是 React 的开发检查模式，用来帮助发现潜在副作用问题。

## PDF.js Worker 配置

`react-pdf` 依赖 `PDF.js worker` 在浏览器后台解析 PDF。

当前配置在 `App.tsx` 顶部：

```tsx
pdfjs.GlobalWorkerOptions.workerSrc = new URL(
  "pdfjs-dist/build/pdf.worker.min.mjs",
  import.meta.url,
).toString();
```

注意：`react-pdf` 官方要求 worker 配置和 `<Document>/<Page>` 的使用位于同一模块，避免模块执行顺序覆盖配置。

## 视图切换

当前视图类型：

```ts
type AppView = "files" | "reader" | "settings";
```

状态：

```ts
const [currentView, setCurrentView] = useState<AppView>("files");
```

三个视图：

- `files`：文件页。
- `reader`：阅读页。
- `settings`：设置页。

相关函数：

- `openReader(document)`：打开指定文档阅读器。
- `closeReader()`：从阅读器回到文件页，但保留当前打开文档。
- `openSettings()`：打开设置页。
- `returnToReader()`：从设置页返回阅读器。
- `clearActiveDocument()`：关闭当前文档并清空阅读器状态。

## 主要 TypeScript 类型

`TypeScript` 类型用于描述数据结构。它不会直接出现在浏览器运行时，但能在开发时发现字段拼写错误和类型错误。

主要类型：

- `DocumentItem`：对应后端 `DocumentItem`。
- `PageItem`：对应后端页面数据。
- `NoteBlockItem`：讲稿文字块。
- `ChatMessageItem`：当前页问答消息。
- `DocumentStatusResponse`：文档状态接口响应。
- `LLMConfigResponse`：LLM 配置接口响应。
- `PageChatResponse`：当前页问答接口响应。
- `DocumentActionState`：文档操作中的忙碌状态。
- `NoteBlockInteraction`：拖动或缩放文字块时的临时交互状态。

当前前端类型与后端返回结构手动同步。未来如果后端 schema 变复杂，可以考虑自动生成 API 类型。

## 文件页状态

文件页主要状态：

- `connectionState`：后端健康检查状态。
- `message`：后端连接提示。
- `selectedFile`：当前选择的 PDF 文件。
- `uploadState`：上传状态。
- `uploadMessage`：上传提示。
- `uploadedDocumentId`：最近上传成功的文档 ID。
- `documents`：文档列表。
- `documentsMessage`：文档列表加载提示。
- `documentStatusById`：按文档 ID 缓存生成进度。
- `expandedDocumentIds`：哪些文档卡片已展开。
- `expandedLectureNotesDocumentId`：哪个文档正在展开逐页讲稿。
- `pagesByDocument`：按文档 ID 缓存页面数据。
- `documentActionState`：重命名、删除、重新生成等操作的忙碌状态。

文件页功能：

- 上传 PDF。
- 展示文档列表。
- 展示 AI 生成进度。
- 展开文档详情。
- 查看课程简介。
- 查看逐页讲稿。
- 重新生成课程简介。
- 重新生成全部讲稿。
- 重新生成单页讲稿。
- 暂停或继续讲稿生成。
- 重命名文档。
- 删除文档。
- 打开阅读器。

## 设置页状态

设置页主要状态：

- `llmConfigState`：加载、保存、测试状态。
- `llmBaseUrl`：LLM 服务地址。
- `llmApiKey`：本次输入的新 API Key。
- `llmApiKeyConfigured`：后端是否已保存 API Key。
- `llmApiKeyPreview`：后端返回的 API Key 掩码。
- `shouldClearLlmApiKey`：用户是否明确清空密钥。
- `llmModel`：模型名。
- `llmTimeoutSeconds`：超时时间。
- `courseSummaryPrompt`：课程简介 prompt。
- `lectureNotesPrompt`：逐页讲稿 prompt。
- `pageChatPrompt`：当前页问答 prompt。
- `isCourseSummaryPromptExpanded`：课程简介 prompt 是否展开。
- `isLectureNotesPromptExpanded`：逐页讲稿 prompt 是否展开。
- `isPageChatPromptExpanded`：当前页问答 prompt 是否展开。
- `llmTestPrompt`：测试 prompt。
- `llmTestAnswer`：测试返回。

设置页 API：

- `loadLlmConfig()` -> `GET /api/llm/config`
- `saveLlmConfig()` -> `PATCH /api/llm/config`
- `testLlmConfig()` -> `POST /api/llm/test`

API Key 规则：

- 输入框为空时，保存操作默认不发送 `api_key`，后端保留旧值。
- 勾选清空密钥时，发送 `api_key: ""`。
- 输入新密钥时，发送新值覆盖旧值。

## 阅读器状态

阅读器主要状态：

- `activeReaderDocument`：当前打开的文档。
- `readerState`：PDF 加载状态。
- `readerMessage`：PDF 或阅读器提示。
- `currentPdfPage`：当前页码，从 1 开始。
- `totalPdfPages`：PDF.js 读取出的总页数。
- `isEditingPdfPage`：页码是否处于输入状态。
- `pdfPageInputValue`：页码输入框草稿。
- `readerViewportWidth`：PDF 渲染可用宽度。
- `readerWorkspaceSize`：阅读工作区宽高。
- `pdfPageNaturalSize`：当前 PDF 页原始宽高。
- `thumbnailSidebarWidth`：缩略图侧栏宽度。
- `courseSummarySidebarWidth`：右侧栏宽度。
- `readerRightSidebar`：右侧栏显示课程简介、聊天或关闭。
- `isReaderTopbarCollapsed`：顶部工具栏是否折叠。
- `isReaderChatCollapsed`：底部聊天栏是否折叠。

阅读器布局模仿 PowerPoint 的工作区：

- 顶部工具栏。
- 状态条。
- 中间 PDF 工作区。
- 左侧缩略图栏。
- 中间 PDF 当前页。
- 可选右侧栏。
- 底部问答栏。

## PDF 展示

阅读器用 `react-pdf` 的组件展示 PDF：

- `<Document>` 加载原始 PDF。
- `<Page>` 展示当前页。
- `<Thumbnail>` 展示缩略图。

PDF 文件 URL：

```text
/api/documents/{document_id}/file
```

加载成功后：

- `handlePdfLoadSuccess({ numPages })` 保存总页数。
- `readerState = ready`。

加载失败后：

- `readerState = error`。
- `readerMessage` 显示错误。

页面切换入口：

- 上一页、下一页按钮。
- 点击缩略图。
- 点击页码进入输入状态后提交页码。

页码边界：

- 小于 1 时跳到第一页。
- 大于总页数时跳到最后一页。

## 状态轮询

前端通过 `loadDocumentStatus(documentId)` 请求：

```text
GET /api/documents/{document_id}/status
```

`mergeDocumentStatus(statusData)` 会把状态接口返回合并到：

- `documentStatusById`
- `documents`
- `pagesByDocument`

轮询由 `shouldPollDocumentStatus(statusData)` 判断，实际依赖后端返回的 `should_poll`。

当状态更新的文档正处于阅读器或展开讲稿面板中，前端会顺便刷新页面列表，让讲稿正文、文字块和问答历史保持更新。

## 讲稿文字块交互

讲稿文字块使用前端自定义 pointer 事件实现拖动和缩放，没有使用 `react-rnd`。

核心状态：

- `draftNoteBlockLayouts`：拖动或缩放过程中的临时布局。
- `noteBlockInteraction`：当前正在进行的拖动或缩放交互。

核心函数：

- `startNoteBlockDrag(...)`
- `startNoteBlockResize(...)`
- `buildDraggedNoteBlockLayout(...)`
- `buildResizedNoteBlockLayout(...)`
- `clampNoteBlockLayout(...)`
- `saveNoteBlockPosition(...)`

交互流程：

1. 用户按下文字块标题栏或缩放手柄。
2. 前端记录起始鼠标位置和起始布局。
3. `window.pointermove` 不断计算临时布局。
4. `window.pointerup` 计算最终布局。
5. 前端先乐观更新本地缓存。
6. 调用 `PATCH /api/note-blocks/{note_block_id}` 保存。
7. 成功后用后端返回的最终值覆盖本地缓存。

最小尺寸：

- 前端：`MIN_NOTE_BLOCK_WIDTH = 120`
- 前端：`MIN_NOTE_BLOCK_HEIGHT = 80`
- 后端也会再次校验相同限制。

## 当前页问答

当前页问答状态：

- `pageQuestionInput`：输入框内容。
- `submittingPageChatId`：正在等待回答的页面 ID。
- `pageChatMessage`：问答状态提示或错误。

提交函数：

```ts
submitPageQuestion(page)
```

请求：

```text
POST /api/pages/{page_id}/chat
```

成功后：

- 更新当前页的 `chat_messages`。
- 清空输入框。
- 显示“AI 老师已回答。”

失败后：

- 显示错误。
- 重新加载页面列表，因为后端可能已经保存了用户问题。

## 当前前端限制

- `App.tsx` 文件过大，后续维护成本会升高。
- 没有独立 API client 层，所有 `fetch` 调用写在组件内部。
- Markdown 只是按文本展示，还没有真正渲染标题、列表、代码块。
- 没有 LaTeX 渲染。
- 没有前端自动测试。
- 拖动文字块不会限制在 PDF 工作区内，只保证最小尺寸和数字有效。

## 推荐拆分方向

建议后续按下面顺序拆：

```text
frontend/src/
  api/
    client.ts
    documents.ts
    llm.ts
  types/
    api.ts
  components/
    FileManager/
    Settings/
    Reader/
    NoteBlock/
    PageChat/
  hooks/
    useDocumentPolling.ts
    usePdfReader.ts
    useNoteBlockInteraction.ts
```

拆分原则：

- 先抽类型和 API client，因为它们最容易稳定复用。
- 再抽无状态展示组件。
- 最后抽复杂 hook。
- 拆分过程中不要改变接口字段和用户行为，除非有测试覆盖。
