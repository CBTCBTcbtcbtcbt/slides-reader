# 前端架构

## 总体说明

前端已经从单个大组件拆分为 API、类型、组件、hook 和样式模块。拆分目标是保持现有用户行为不变，同时降低后续开发 Markdown/LaTeX、Prompt 管理和多样标注功能时的维护成本。

`frontend/src/App.tsx` 现在仍然是顶层协调组件，但不再直接承载所有 JSX、`fetch` 和交互细节。

## 入口文件

`main.tsx` 负责把 `App` 挂载到页面，并用 `BrowserRouter` 提供前端路由能力：

```tsx
ReactDOM.createRoot(document.getElementById("root") as HTMLElement).render(
  <React.StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </React.StrictMode>,
);
```

`StrictMode` 是 React 的开发检查模式，用来帮助发现潜在副作用问题。

`BrowserRouter` 是 React Router 提供的路由组件。它使用浏览器真实 URL，例如 `/settings` 和 `/documents/{document_id}/read?page=3`，因此刷新页面后仍能从 URL 恢复当前页面。

## 前端路由

当前前端路由如下：

| 路径 | 页面 | 说明 |
| --- | --- | --- |
| `/` | 文件页 | 上传、选择、重命名和删除 slides 文件 |
| `/settings` | 设置页 | 配置 OpenAI-compatible LLM 和 prompt |
| `/documents/:documentId/read?page=3` | 阅读页 | 打开指定文档，并读取 query string 中的 `page` 作为当前 PDF 页码 |

这里的“路由”只控制前端显示哪个页面，不改变后端接口。后端 API 仍然统一使用 `/api/...` 路径，例如 `GET /api/documents` 和 `GET /api/documents/{document_id}/pages`。

阅读页刷新恢复流程：

- `App.tsx` 从 URL path 读取 `documentId`。
- `App.tsx` 从 query string 读取 `page`。
- 前端重新加载文档列表，再用 `documentId` 找到对应文档。
- 找到文档后重新加载页面数据和 PDF。
- `page` 不是正整数时自动修正为 `1`。
- `page` 超过 PDF 总页数时，PDF 加载成功后自动修正为最后一页。
- 文档不存在或已被删除时，清空阅读器状态并跳回 `/`。

翻页、点击缩略图和页码输入都会更新 URL 中的 `page`。这些页内跳转使用 `replace: true`，避免浏览器历史记录被每次翻页塞满；文件页、设置页和阅读页之间的切换保留正常历史记录，浏览器前进和后退可以按页面切换。

## 生产部署 Fallback

`BrowserRouter` 使用真实路径，所以生产环境直接访问 `/documents/{document_id}/read?page=3` 时，静态服务器必须返回前端的 `index.html`，再由 React Router 接管页面渲染。

生产服务器需要遵守两条规则：

- `/api/` 路径转发到 FastAPI 后端。
- 非 `/api/` 的前端路径回退到 `index.html`。

Nginx 示例：

```nginx
location /api/ {
  proxy_pass http://127.0.0.1:8000;
}

location / {
  try_files $uri $uri/ /index.html;
}
```

如果没有这类 SPA fallback，开发环境可能正常，但生产环境刷新 `/settings` 或 `/documents/{document_id}/read?page=3` 会由服务器直接查找同名文件，最终返回 404。

## App 的职责

`App.tsx` 保留顶层状态和跨视图协调：

- 根据 URL 决定当前页面：文件页、阅读页或设置页。
- 后端健康检查状态。
- 文档列表、当前打开文档和生成进度缓存。
- LLM 配置表单状态。
- PDF 阅读器页码、右侧栏、顶部栏和底部问答栏状态。
- 调用 API 模块并把结果合并到本地状态。
- 把状态和回调传给拆出的组件。

后续如果继续拆分，可以优先把部分状态逻辑再移动到专门 hook，但不要在没有测试保护时改变接口字段或用户行为。

## 类型模块

`types/api.ts` 保存后端响应和请求 payload 类型，例如：

- `DocumentItem`
- `PageItem`
- `NoteBlockItem`
- `ChatMessageItem`
- `DocumentStatusResponse`
- `LLMConfigResponse`
- `PageChatResponse`

`types/ui.ts` 保存前端内部状态类型，例如：

- `ReaderState`
- `ReaderRightSidebar`
- `DocumentActionState`
- `NoteBlockLayout`
- `NoteBlockInteraction`

这样做可以把“后端接口结构”和“前端界面状态”分开，避免后续维护时混在一起。

## API 模块

前端所有 `fetch` 调用集中在 `api/`：

- `http.ts`：封装基础请求、JSON 解析和错误信息读取。
- `documents.ts`：文档、页面、讲稿、文字块和当前页问答接口。
- `llm.ts`：健康检查、LLM 配置读取、保存和测试接口。

`http.ts` 中的 `requestJson<T>()` 会统一处理非 2xx HTTP 状态码。FastAPI 错误响应通常包含 `detail` 字段，前端优先展示这个字段。

## 组件拆分

### FilesView

`components/FilesView/FilesView.tsx` 负责文件页 JSX：

- 上传 PDF/PPT/PPTX。
- 展示文档列表。
- 展示课程简介和逐页讲稿进度。
- 展开文档详情。
- 重命名、删除、暂停/继续讲稿生成。
- 触发课程简介、整份讲稿和单页讲稿重新生成。
- 打开阅读器。

它接收 `App.tsx` 传入的状态和回调，不直接访问后端。

### SettingsView

`components/SettingsView/SettingsView.tsx` 负责 LLM 配置表单：

- `LLM_BASE_URL`
- `LLM_API_KEY`
- `LLM_MODEL`
- 请求超时时间
- 课程简介 prompt
- 逐页讲稿 prompt
- 当前页问答 prompt
- 测试 prompt 和测试回答

API Key 规则保持不变：

- 输入框为空时保存，不发送 `api_key`，后端保留旧值。
- 勾选清空密钥时，发送 `api_key: ""`。
- 输入新密钥时，发送新值覆盖旧值。

### ReaderView

`components/ReaderView/ReaderView.tsx` 负责 PDF 阅读器 JSX：

- 顶部工具栏。
- 生成状态提示。
- PDF 主工作区。
- 左侧缩略图栏。
- 右侧课程简介或当前页问答侧栏。
- 底部当前页问答栏。

`react-pdf` 的 `pdfjs.GlobalWorkerOptions.workerSrc` 也放在这个模块里。`react-pdf` 要求 worker 配置和 `<Document>/<Page>/<Thumbnail>` 使用位于同一模块，避免模块执行顺序覆盖配置。

### NoteBlock

`components/NoteBlock/NoteBlock.tsx` 负责讲稿文字块：

- 显示当前页讲稿内容。
- 提供拖动手柄。
- 提供八个方向的缩放手柄。
- 触发单页讲稿重新生成。

拖拽和缩放的计算逻辑不在组件里，而是在 `useNoteBlockInteraction.ts` 中。

### PageChat

`components/PageChat/PageChat.tsx` 负责当前页问答内容：

- 显示当前页问答历史。
- 区分学生和 AI 老师消息。
- 提供问题输入框。
- 显示问答状态或错误。

问答历史按页面隔离，由后端 `page_id` 关联。

### MarkdownContent

`components/MarkdownContent.tsx` 负责统一渲染 LLM 输出的 Markdown 和 LaTeX 公式。

它使用：

- `react-markdown`：把 Markdown 字符串转换成 React 组件。
- `remark-gfm`：支持 GitHub Flavored Markdown，例如表格、任务列表和删除线。
- `remark-math`：识别 `$...$` 和 `$$...$$` 公式。
- `rehype-katex` 和 `katex`：把公式渲染成浏览器可显示的数学排版。

安全规则：

- 开启 `skipHtml`，不渲染 Markdown 中的 HTML。
- 不使用 `dangerouslySetInnerHTML`。
- 不引入 `rehype-raw`。
- 链接使用 `target="_blank"` 和 `rel="noreferrer"`。

当前使用位置：

- 文件页课程简介。
- 文件页逐页讲稿列表。
- 阅读器右侧课程简介。
- 阅读器 PDF 页面上的讲稿浮层。
- 当前页问答历史消息。
- 设置页 LLM 测试回答。

## Hook 拆分

### useDocumentPolling

`hooks/useDocumentPolling.ts` 封装文档状态轮询。

它根据：

- 文档列表中的粗略状态。
- `DocumentStatusResponse.should_poll`。
- 轮询间隔 `DOCUMENT_STATUS_POLL_INTERVAL_MS`。

决定哪些文档需要继续请求 `GET /api/documents/{document_id}/status`。

### usePdfSizing

`hooks/usePdfSizing.ts` 封装 PDF 阅读器尺寸计算：

- 阅读器可用宽高。
- 缩略图侧栏宽度。
- 右侧栏宽度。
- 左右侧栏拖动调整。
- 窄屏布局下的工作区尺寸扣除。

这让 `ReaderView` 只负责渲染，`App.tsx` 只负责保存 PDF 页码和加载状态。

### useNoteBlockInteraction

`hooks/useNoteBlockInteraction.ts` 封装讲稿文字块拖动和缩放：

- 保存拖动过程中的临时布局。
- 监听 `pointermove` 和 `pointerup`。
- 计算拖动后的坐标。
- 计算缩放后的坐标和宽高。
- 保证最小宽度 `120`、最小高度 `80`。
- 鼠标松开后调用后端保存接口。

## 样式结构

`styles.css` 现在只作为样式入口，按模块导入实际 CSS：

```css
@import "./styles/base.css";
@import "./styles/files.css";
@import "./styles/settings.css";
@import "./styles/reader.css";
@import "./styles/note-block.css";
@import "./styles/page-chat.css";
@import "./styles/markdown.css";
@import "./styles/responsive.css";
```

各文件职责：

| 文件 | 职责 |
| --- | --- |
| `base.css` | 全局变量、页面壳、通用按钮、连接状态、通用状态 badge |
| `files.css` | 上传区域、文档列表、课程简介盒子、逐页讲稿列表、重命名和删除 |
| `settings.css` | LLM 配置表单、prompt 折叠区、测试回答 |
| `reader.css` | 阅读器整体、顶部栏、PDF 工作区、缩略图、右侧栏、底部问答停靠区 |
| `note-block.css` | 讲稿文字块、拖动手柄、缩放手柄 |
| `page-chat.css` | 当前页问答历史、输入框、状态提示 |
| `markdown.css` | Markdown 正文、表格、代码块和 KaTeX 公式样式 |
| `responsive.css` | `920px` 和 `560px` 断点下的响应式布局 |

类名尽量沿用原有 CSS，避免视觉变化。

## 当前限制

- 前端还没有自动化测试，当前只通过 `npm run build` 验证 TypeScript 和生产构建。
- `App.tsx` 仍然承担较多全局状态协调，后续可以继续按风险拆分。
- 拖动文字块目前只保证最小尺寸和数值有效，还没有限制在 PDF 工作区内。
