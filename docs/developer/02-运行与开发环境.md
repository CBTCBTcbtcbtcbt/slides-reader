# 运行与开发环境

## 环境要求

当前项目在 Windows + PowerShell 环境下开发。建议使用：

- Python 3.11 或更新版本。
- Node.js 20 或更新版本。
- npm。
- PowerShell。
- LibreOffice，用于把 PPT/PPTX 转换为 PDF。

`Python` 用来运行 FastAPI 后端。`Node.js` 和 `npm` 用来运行 Vite 前端。`PowerShell` 是 Windows 上的命令行环境，本项目的 `launch.ps1` 使用 PowerShell 脚本。`LibreOffice` 是开源办公套件，本项目使用它的 `soffice.exe` 命令行入口把 PPT/PPTX 转换为 PDF。

如果本机没有 LibreOffice，可以在项目根目录运行：

```powershell
.\setup-env.ps1
```

这个脚本会优先查找本机已安装的 LibreOffice；如果找不到，会下载 LibreOffice Portable 到 `tools/libreoffice/`。`Portable` 表示便携版，不需要写入系统安装目录。

## 后端安装

进入后端目录：

```powershell
cd backend
```

创建虚拟环境：

```powershell
python -m venv .venv
```

`venv` 是 Python 自带的虚拟环境工具。虚拟环境可以让这个项目的 Python 依赖和系统里其他项目隔离。

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

安装依赖：

```powershell
pip install -r requirements.txt
```

当前后端依赖包括：

- `fastapi`：Web API 框架。
- `uvicorn[standard]`：运行 FastAPI 的 ASGI 服务器。`ASGI` 是 Python Web 服务和异步应用之间的接口规范。
- `PyMuPDF`：PDF 解析和渲染库。
- `python-multipart`：让 FastAPI 支持 `multipart/form-data` 文件上传。

启动后端：

```powershell
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

`--reload` 表示代码变更后自动重启服务，适合开发环境。

## 前端安装

进入前端目录：

```powershell
cd frontend
```

安装依赖：

```powershell
npm install
```

启动前端：

```powershell
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

当前前端依赖包括：

- `react`：构建页面组件。
- `react-dom`：把 React 组件挂载到浏览器 DOM。
- `react-pdf`：在浏览器中展示 PDF。
- `typescript`：提供类型检查。
- `vite`：开发服务器和构建工具。

## 一键启动

项目根目录提供 `launch.ps1`：

```powershell
.\launch.ps1
```

这个脚本会检查：

- `backend/.venv/Scripts/python.exe` 是否存在。
- `frontend/node_modules` 是否存在。
- LibreOffice 的 `soffice.exe` 是否存在。

如果依赖已经安装，它会分别打开两个 PowerShell 窗口：

- 后端：`http://127.0.0.1:8000`
- 前端：`http://localhost:5173`

注意：脚本当前会使用可见窗口启动前后端，适合本地开发观察日志。
如果找不到 LibreOffice，`launch.ps1` 会提示先运行 `.\setup-env.ps1`，而不会在启动时自动下载大文件。

## Vite 代理

前端开发服务器在 `frontend/vite.config.ts` 中配置了 `/api` 代理：

```ts
proxy: {
  "/api": {
    target: "http://127.0.0.1:8000",
    changeOrigin: true,
  },
}
```

这表示前端代码只需要请求 `/api/...`，Vite 会在开发环境把请求转发给 FastAPI 后端。这样前端不需要写死完整后端地址，也可以避免浏览器跨域限制。

## 运行数据目录

默认运行数据目录：

```text
storage/
```

目录内容：

```text
storage/app.db
storage/documents/{document_id}.pdf
storage/conversion-tmp/
storage/libreoffice-profile/
storage/pages/{document_id}-page-{page_number}.png
```

可以通过环境变量覆盖：

```powershell
$env:SLIDES_READER_STORAGE_DIR="C:\temp\slides-reader-storage"
```

这个环境变量在测试时尤其重要。测试应该指向临时目录，而不是项目实际开发目录。

## LLM 配置来源

后端默认从环境变量读取 LLM 初始配置：

```powershell
$env:LLM_BASE_URL="https://api.openai.com/v1"
$env:LLM_API_KEY="你的 API Key"
$env:LLM_MODEL="gpt-4.1-mini"
$env:LLM_TIMEOUT_SECONDS="60"
```

也可以设置默认 prompt：

```powershell
$env:COURSE_SUMMARY_PROMPT="生成课程简介的 prompt"
$env:LECTURE_NOTES_PROMPT="生成逐页讲稿的 prompt"
$env:PAGE_CHAT_PROMPT="当前页问答的 prompt"
```

环境变量只作为初始默认值。一旦用户在 WebUI 保存配置，数据库 `app_settings` 表中的配置优先级更高。

## 健康检查

后端启动后可以访问：

```text
GET http://127.0.0.1:8000/api/health
```

成功响应：

```json
{
  "status": "ok",
  "service": "slides-reader-api"
}
```

前端启动后也会自动请求这个接口。如果失败，文件列表和 LLM 配置不会正常加载。

## 构建检查

前端构建：

```powershell
cd frontend
npm run build
```

这个命令会先执行 TypeScript 构建，再由 Vite 打包。它可以发现很多类型错误和 JSX 语法错误。

后端目前没有独立测试目录。最基础的后端语法检查可以运行：

```powershell
cd backend
.\.venv\Scripts\python.exe -m py_compile main.py
```

`py_compile` 只检查 Python 文件能否编译，不会启动服务，也不会覆盖业务流程测试。
