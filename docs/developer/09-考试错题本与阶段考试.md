# 考试、错题本与阶段考试

本文档说明当前已经实现的考试、错题本和阶段考试功能。这里记录的是代码现状。

## 功能入口

前端有三类考试相关页面：

- 文件页：在每份课件下生成普通试卷、查看试卷状态、开始答题和删除试卷。
- 错题本页：查看答错题目、标记已复习和移除错题。
- 阶段考试创建页：选择多份课件，创建综合阶段考试。

普通试卷和阶段考试都使用后端 `exams` 表保存实际题目。阶段考试额外使用 `phase_exams` 表保存名称、参与课件、难度和关联试卷 ID。

## 普通试卷生成

普通试卷基于单份课件生成。入口接口是：

```text
POST /api/documents/{document_id}/exams/generate
```

请求体：

```json
{
  "difficulty": "medium"
}
```

生成流程：

1. 后端先创建一条 `exams` 记录，状态为 `pending`。
2. 后端在线程中异步生成题目。
3. 生成任务开始后把状态改为 `processing`。
4. LLM 返回题目 JSON 后，后端校验并写入 `exam_questions`。
5. 写入成功后把试卷状态改为 `ready`。
6. 如果 LLM 调用、JSON 解析或写入失败，状态改为 `failed`，错误写入 `error_message`。

前端文件页会轮询 `pending` 和 `processing` 状态的试卷列表，避免用户必须手动刷新。

## 题型

当前支持两类题目：

- `choice`：单选题。
- `fill_in`：填空题，当前主要用于数字答案。

题目字段保存在 `exam_questions` 表，包含题干、选项、正确答案、解析、分值、来源页码、难度和知识点标签。

答题页读取题目时使用：

```text
GET /api/exams/{exam_id}/questions?include_answer=false
```

`include_answer=false` 会隐藏 `answer` 和 `explanation`，避免答题前暴露答案。

## 答题和判分

答题页提交接口：

```text
POST /api/exams/{exam_id}/attempts
```

请求体是题目 ID 到用户答案的映射：

```json
{
  "answers": {
    "question-id": "A"
  }
}
```

后端行为：

1. 创建 `exam_attempts` 答题记录。
2. 逐题比较用户答案和正确答案。
3. 计算本次得分。
4. 把答错题目写入 `wrong_questions`。
5. 返回完整判分结果。

结果页路由包含 `attempt_id`，刷新页面后会调用：

```text
GET /api/exams/{exam_id}/attempts/{attempt_id}/result
```

因此结果页不依赖提交后留在前端内存里的临时结果。

## 错题本

错题本保存用户答错的题目记录。相关接口：

```text
GET /api/wrong-questions
GET /api/documents/{document_id}/wrong-questions
POST /api/wrong-questions/{wrong_id}/review
DELETE /api/wrong-questions/{wrong_id}
GET /api/wrong-questions/statistics?document_id={document_id}
```

当前行为：

- 答错题目会写入 `wrong_questions`。
- 错题本展示原题、用户答案、正确答案、解析和知识点标签。
- 用户可以把错题标记为已复习。
- 用户可以从错题本移除某条记录。
- 删除错题只删除 `wrong_questions` 记录，不删除原试卷题目和答题记录。

错题统计接口会按知识点统计错题，用于阶段考试生成时加权。

## 阶段考试

阶段考试基于多份课件生成综合试卷。相关接口：

```text
GET /api/phase-exams
POST /api/phase-exams/generate
GET /api/phase-exams/{phase_exam_id}
DELETE /api/phase-exams/{phase_exam_id}
```

创建阶段考试的请求体：

```json
{
  "document_ids": ["document-id-1", "document-id-2"],
  "name": "期中复习卷",
  "difficulty": "medium"
}
```

生成流程：

1. 后端创建 `phase_exams` 记录，状态为 `pending`。
2. 后端创建一条普通 `exams` 记录作为题目容器。
3. 后端立即把普通试卷 ID 写回 `phase_exams.exam_id`。
4. 后端在线程中生成综合题目。
5. 生成成功后，关联普通试卷和阶段考试都进入 `ready`。
6. 生成失败时，关联普通试卷和阶段考试都进入 `failed`。

前端开始阶段考试时，实际跳转到关联普通试卷的答题页：

```text
/exams/{exam_id}/take
```

这样阶段考试可以复用普通试卷的答题、判分、结果页和错题本链路。

## 删除规则

删除普通试卷时，后端按顺序清理：

```text
wrong_questions -> exam_attempts -> exam_questions -> exams
```

删除阶段考试时，后端会先读取 `phase_exams.exam_id`。如果存在关联普通试卷，就按普通试卷删除规则清理题目、答题记录和错题，再删除 `phase_exams` 记录。

删除文档时，后端会清理：

- 该文档直接关联的普通试卷。
- `document_ids` 中包含该文档的阶段考试。
- 上述试卷关联的题目、答题记录和错题。

这些规则保证删除文档、试卷或阶段考试后，不会留下无法从 UI 清理的孤儿数据。

## 前端状态管理

考试相关状态集中在：

```text
frontend/src/hooks/useExamWorkflows.ts
```

这个 hook 负责：

- 按文档加载普通试卷列表。
- 加载阶段考试列表。
- 加载错题本。
- 生成和删除普通试卷。
- 创建和删除阶段考试。
- 进入答题页。
- 提交答题。
- 刷新结果页。
- 标记和删除错题。
- 轮询生成中的普通试卷和阶段考试。

`App.tsx` 保留顶层路由、文档和阅读器协调职责，通过 `useExamWorkflows` 消费考试相关状态和操作。

## 测试覆盖

后端 API 契约测试覆盖考试相关数据清理和状态行为。

前端测试覆盖：

- `useExamWorkflows` 中生成中状态判断。
- 文档删除后考试缓存清理。
- `PhaseExamCreateView` 中选择课件、填写名称、选择难度和提交创建阶段考试。

运行命令：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest

cd ..\frontend
npm run test
```
