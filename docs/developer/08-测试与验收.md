# 测试与验收

## 当前测试状态

当前仓库还没有正式的自动测试目录。任务文档中已经规划过后端测试、前端测试和手动验收，但代码库中尚未落地完整测试体系。

因此后续任何较大改动都建议先补测试，再修改业务逻辑。尤其是下面这些功能：

- PDF 上传校验。
- SQLite 数据写入。
- PDF 页数和页面记录。
- LLM 配置保存。
- 课程简介和讲稿状态流转。
- 当前页问答保存用户问题。
- 文档删除的级联清理。
- 前端阅读器翻页和讲稿文字块保存。

## 基础检查命令

后端语法检查：

```powershell
cd backend
.\.venv\Scripts\python.exe -m py_compile main.py
```

前端构建检查：

```powershell
cd frontend
npm run build
```

这些命令只能证明代码能编译或构建，不能证明业务流程正确。

## 建议的后端测试技术

建议使用：

- `pytest`
- `fastapi.testclient.TestClient`
- 临时 `SLIDES_READER_STORAGE_DIR`
- LLMClient mock

`pytest` 是 Python 常用测试框架。`TestClient` 是 FastAPI 提供的测试客户端，可以不真正启动网络端口就调用 API。

LLM 相关测试不要真实请求外部模型。应该用 mock 替代 LLMClient，让测试结果稳定、不依赖网络、不消耗 API Key。

## 后端测试建议覆盖

### 健康检查

目标：

- `GET /api/health` 返回 `200`。
- 返回 `status = ok`。

### PDF 上传成功

目标：

- 上传合法 PDF 返回 `201`。
- 响应包含 `document_id`。
- `storage/documents/{document_id}.pdf` 存在。
- 数据库 `documents` 有记录。
- 数据库 `pages` 有对应页面记录。

### 非 PDF 被拒绝

目标：

- 上传 `.txt` 或错误 `content-type` 返回 `400`。
- 不写入 `documents` 记录。
- 不保存文件。

### 损坏 PDF

目标：

- 上传损坏 PDF 不导致服务崩溃。
- 文档状态变为 `failed`。
- `error_message` 有内容。
- 课程简介状态变为 `failed`。

### 页面列表

目标：

- `GET /api/documents/{document_id}/pages` 返回页码从 1 开始。
- 返回 `text`、`image_url`、`lecture_notes_status`。
- 空白页也有记录。

### LLM 配置

目标：

- 保存配置后再次读取能看到新配置。
- 不返回 API Key 明文。
- `api_key_preview` 是掩码。
- 不传 `api_key` 时保留旧值。
- 传空字符串时清空旧值。
- 超时时间小于 5 或大于 300 返回 `400`。

### 课程简介生成

使用 mock LLM 返回固定文本。

目标：

- 生成成功后 `course_summary_status = ready`。
- `course_summary` 保存 mock 文本。
- 生成失败后 `course_summary_status = failed`。
- 失败不删除页面记录。

### 逐页讲稿生成

使用 mock LLM。

目标：

- 课程简介未 ready 时重新生成讲稿返回 `400`。
- 生成成功后每页 `lecture_notes_status = ready`。
- 某页失败时该页 `failed`，其他页继续。
- 生成成功后自动创建或更新 `note_blocks`。
- 已有文字块时重新生成只更新内容，不覆盖位置。

### 暂停和继续

目标：

- 暂停接口把 `lecture_notes_paused` 设为 `true`。
- 继续接口把 `lecture_notes_paused` 设为 `false`。
- 继续生成不覆盖已经 `ready` 的页面。

### 当前页问答

使用 mock LLM。

目标：

- 提交问题后保存 `user` 消息和 `assistant` 消息。
- 返回完整消息列表。
- LLM 失败时用户问题仍保存在数据库中。
- 空问题返回 `400`。

### 文档删除

目标：

- 删除文档返回 `204`。
- 删除 `documents` 记录。
- 删除该文档所有 `pages`。
- 删除相关 `note_blocks`。
- 删除相关 `chat_messages`。
- 删除本地 PDF。
- 删除页面截图。
- 本地文件已经不存在时，数据库仍然删除成功。

## 建议的前端测试技术

建议使用：

- `Vitest`
- `@testing-library/react`
- `@testing-library/user-event`
- `msw` 或手写 `fetch` mock

`Vitest` 是适合 Vite 项目的测试框架。`Testing Library` 是测试 React 组件用户行为的工具。`msw` 是 Mock Service Worker，可以模拟后端 API。

## 前端测试建议覆盖

### 后端连接

目标：

- `/api/health` 成功时显示连接成功。
- 失败时显示错误提示。

### 文件上传

目标：

- 选择非 PDF/PPT/PPTX 时显示错误。
- 选择 PDF/PPT/PPTX 后可提交。
- 上传成功后显示成功消息。
- 上传成功后刷新文档列表。

### LLM 设置

目标：

- 配置加载后表单填充。
- API Key 不显示明文。
- prompt 默认折叠。
- 点击展开后显示完整 textarea。
- 保存时按规则发送或不发送 `api_key`。
- 测试成功后显示模型回答。

### 文档列表

目标：

- 展示文档标题、状态、页数。
- 展开后显示课程简介和讲稿状态。
- 点击重命名进入编辑状态。
- 空标题不提交。
- 删除前调用确认框。

### 阅读器

目标：

- 点击“阅读 slides”后进入阅读器。
- PDF 加载成功后显示页码和翻页按钮。
- 上一页和下一页按钮有边界限制。
- 点击缩略图切页。
- 页码输入能跳页。

### 讲稿文字块

目标：

- 当前页讲稿 ready 且有 note block 时显示文字块。
- 拖动结束后调用 `PATCH /api/note-blocks/{id}`。
- 缩放后不会小于最小尺寸。

### 当前页问答

目标：

- 空问题不提交。
- 提交问题时显示等待状态。
- 成功后显示新消息并清空输入框。
- 失败后显示错误，并重新加载页面数据。

## 手动验收流程

以下流程用于没有自动测试时的完整人工确认。

### 准备

1. 清空或备份 `storage/`。
2. 启动后端。
3. 启动前端。
4. 准备一份 3 页以上 PDF。
5. 在设置页配置可用的 LLM 服务。

### 基础连接

1. 打开 `http://localhost:5173`。
2. 确认页面显示后端连接成功。
3. 打开设置页。
4. 点击测试连接。
5. 确认能显示模型回答。

### 上传与解析

1. 回到文件页。
2. 选择 PDF、PPT 或 PPTX。
3. 点击上传。
4. 确认上传成功。
5. 确认文档列表出现新文档。
6. 确认显示总页数。
7. 打开 `storage/documents/`，确认系统实际使用的 PDF 文件存在。
8. 打开 `storage/pages/`，确认 PNG 截图存在。

### 课程简介和讲稿

1. 观察文档生成进度。
2. 确认课程简介进入生成中。
3. 等待课程简介生成完成。
4. 确认逐页讲稿开始生成。
5. 展开文档详情。
6. 查看课程简介内容。
7. 展开逐页讲稿。
8. 确认每页讲稿状态和正文。
9. 点击重新生成某一页讲稿。
10. 确认该页状态刷新。

### 暂停与继续

1. 上传一份页数较多的 PDF。
2. 在讲稿生成中点击暂停。
3. 确认页面显示已暂停。
4. 等待一段时间，确认后续页不继续大量生成。
5. 点击继续生成。
6. 确认生成恢复。

### 阅读器

1. 点击“阅读 slides”。
2. 确认能看到系统实际使用的 PDF 当前页。
3. 点击下一页和上一页。
4. 点击缩略图切换页面。
5. 点击页码并输入目标页。
6. 确认页码边界正常。
7. 打开课程简介侧栏。
8. 打开当前页问答侧栏。

### 讲稿文字块

1. 切到已经生成讲稿的页面。
2. 确认看到讲稿文字块。
3. 拖动文字块到新位置。
4. 刷新页面或重新打开阅读器。
5. 确认位置被保存。
6. 缩放文字块。
7. 再次刷新，确认尺寸被保存。

### 当前页问答

1. 在当前页问答框输入问题。
2. 点击提交。
3. 确认显示等待状态。
4. 确认出现 AI 老师回答。
5. 切换到另一页，确认不会显示上一页问答历史。
6. 切回原页，确认历史仍存在。

### 重命名和删除

1. 回到文件页。
2. 点击重命名，输入新标题。
3. 保存后确认标题更新。
4. 输入空标题，确认被拒绝。
5. 点击删除。
6. 确认后文档从列表消失。
7. 检查 `storage/documents/` 和 `storage/pages/` 中对应文件已删除。

## 验收通过标准

- 后端不会因为坏 PDF 或 LLM 错误崩溃。
- 前端所有主要操作都有明确状态提示。
- 课程简介和逐页讲稿状态能正确显示。
- 单页失败不会破坏整份文档。
- 当前页问答按页面隔离。
- 重命名和删除行为符合预期。
- 前端构建通过。
