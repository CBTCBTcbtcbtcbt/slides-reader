# 测试与验收

## 当前测试状态

后端已经新增契约测试目录：

```text
backend/tests/test_api_contract.py
```

契约测试关注接口路径、状态码、响应字段和关键副作用，用来确保代码结构拆分后 API 行为不变。

前端当前还没有自动化测试，主要通过 TypeScript 编译和 Vite 生产构建验证。

## 测试依赖

后端开发测试依赖放在：

```text
backend/requirements-dev.txt
```

当前包含：

- `pytest`
- `httpx`

`pytest` 是 Python 常用测试框架。`httpx` 是 FastAPI `TestClient` 需要使用的 HTTP 客户端依赖。

## 后端测试运行命令

如果还没有安装测试依赖：

```powershell
cd backend
.\.venv\Scripts\pip.exe install -r requirements-dev.txt
```

运行后端契约测试：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest
```

测试会使用临时 `SLIDES_READER_STORAGE_DIR`，不会污染项目根目录下的 `storage/`。

## 后端契约测试覆盖

### 健康检查和空文档列表

覆盖：

- `GET /api/health` 返回 `200`。
- 响应包含 `status = ok`。
- 空数据库下 `GET /api/documents` 返回空数组。

### LLM 配置

覆盖：

- 保存配置后可以再次读取。
- API Key 不返回明文。
- `api_key_preview` 返回掩码。
- 不传 `api_key` 时保留旧密钥。
- 传空字符串 `api_key: ""` 时清空密钥。

### PDF 上传、页面、状态、截图和删除

覆盖：

- 上传一页测试 PDF 返回 `201`。
- 创建文档记录。
- 创建页面记录。
- 保存系统实际使用的 PDF。
- 保存页面 PNG 截图。
- `GET /api/documents/{document_id}/pages` 返回页面字段。
- `GET /api/documents/{document_id}/status` 返回状态字段。
- `GET /api/documents/{document_id}/pages/{page_number}/image` 返回 PNG。
- `DELETE /api/documents/{document_id}` 删除数据库记录、本地 PDF 和页面截图。

上传后的课程简介后台任务在测试中 mock 为 no-op，避免真实请求 LLM。

### 当前页问答

覆盖：

- 使用 mock LLM 替代真实模型请求。
- `POST /api/pages/{page_id}/chat` 保存 `user` 消息。
- 同一接口保存 `assistant` 消息。
- 响应返回完整消息列表。

### 讲稿文字块位置

覆盖：

- 宽度或高度小于最小值时返回 `400`。
- 合法坐标和尺寸可以成功保存。
- 响应返回更新后的 `x`、`y`、`width`、`height`。

## 前端构建验证

运行：

```powershell
cd frontend
npm run build
```

这个命令会执行：

- TypeScript 项目构建。
- Vite 生产构建。
- CSS import 和模块打包。

当前构建可能出现 Vite 的 chunk 体积提示。这是构建警告，不代表构建失败。

## 建议补充的后端测试

当前契约测试已经覆盖重构风险最高的路径。后续建议继续补：

- 非 PDF/PPT/PPTX 上传被拒绝。
- 损坏 PDF 不导致后端崩溃。
- 课程简介生成成功和失败状态。
- 整份逐页讲稿生成成功、单页失败继续处理。
- 暂停和继续逐页讲稿生成。
- LLM 超时、网络错误和响应结构异常。

这些测试仍然应该使用 mock LLM，避免依赖外部网络和真实 API Key。

## 建议补充的前端测试

前端后续建议引入：

- `Vitest`
- `@testing-library/react`
- `@testing-library/user-event`
- `msw` 或手写 `fetch` mock

优先覆盖：

- 后端健康检查成功和失败。
- 上传文件格式校验。
- LLM 配置保存时 API Key 的发送规则。
- 文档列表展开、重命名和删除。
- 阅读器翻页和页码输入。
- 讲稿文字块拖动和缩放后的保存请求。
- 当前页问答提交成功和失败。

## 手动验收流程

### 准备

1. 清空或备份 `storage/`。
2. 启动后端。
3. 启动前端。
4. 准备一份 3 页以上 PDF。
5. 在设置页配置可用的 LLM 服务。

### 基础连接

1. 打开 `http://localhost:5173`。
2. 确认页面显示后端连接成功。
3. 打开设置页。
4. 点击测试连接。
5. 确认能显示模型回答。

### 上传与解析

1. 回到文件页。
2. 选择 PDF、PPT 或 PPTX。
3. 点击上传。
4. 确认上传成功。
5. 确认文档列表出现新文档。
6. 确认显示总页数。
7. 打开 `storage/documents/`，确认系统实际使用的 PDF 文件存在。
8. 打开 `storage/pages/`，确认 PNG 截图存在。

### 课程简介和讲稿

1. 观察文档生成进度。
2. 确认课程简介进入生成中。
3. 等待课程简介生成完成。
4. 确认逐页讲稿开始生成。
5. 展开文档详情。
6. 查看课程简介内容。
7. 展开逐页讲稿。
8. 确认每页讲稿状态和正文。
9. 点击重新生成某一页讲稿。
10. 确认该页状态刷新。

### 暂停与继续

1. 上传一份页数较多的 PDF。
2. 在讲稿生成中点击暂停。
3. 确认页面显示已暂停。
4. 等待一段时间，确认后续页不继续大量生成。
5. 点击继续生成。
6. 确认生成恢复。

### 阅读器

1. 点击“阅读 slides”。
2. 确认能看到系统实际使用的 PDF 当前页。
3. 点击下一页和上一页。
4. 点击缩略图切换页面。
5. 点击页码并输入目标页。
6. 确认页码边界正常。
7. 打开课程简介侧栏。
8. 打开当前页问答侧栏。

### 讲稿文字块

1. 切到已经生成讲稿的页面。
2. 确认看到讲稿文字块。
3. 拖动文字块到新位置。
4. 刷新页面或重新打开阅读器。
5. 确认位置被保存。
6. 缩放文字块。
7. 再次刷新，确认尺寸被保存。

### 当前页问答

1. 在当前页问答框输入问题。
2. 点击提交。
3. 确认显示等待状态。
4. 确认出现 AI 老师回答。
5. 切换到另一页，确认不会显示上一页问答历史。
6. 切回原页，确认历史仍存在。

### 试卷、错题本和阶段考试

1. 在设置页展开“试卷生成 prompt”，确认可以保存并重新读取。
2. 回到文件页，选择一份已经有课程简介的文档，点击“生成试卷”。
3. 确认试卷列表出现 `pending` 或 `processing` 状态，并会自动刷新。
4. 试卷变为 `ready` 后，点击“开始答题”。
5. 确认答题页刷新后仍能恢复试卷标题和题目，并且浏览器响应中不包含正确答案和解析。
6. 提交一份包含错误答案的答卷。
7. 确认结果页显示总分、每题得分、正确答案和解析。
8. 刷新结果页，确认仍能通过 `attempt_id` 恢复本次结果。
9. 回到文件页，打开错题本，确认错误题目被记录。
10. 在错题本中标记已复习和移除错题，确认列表更新。
11. 点击顶部“阶段考试”，选择多份课件，提交创建。
12. 回到文件页，确认阶段考试列表出现生成中状态，并会自动刷新。
13. 阶段考试变为 `ready` 后，点击“开始考试”，确认能进入同一答题流程。
14. 删除阶段考试，确认阶段考试从列表消失；它关联的普通试卷、答题记录和错题也应被后端清理。

### 重命名和删除

1. 回到文件页。
2. 点击重命名，输入新标题。
3. 保存后确认标题更新。
4. 输入空标题，确认被拒绝。
5. 点击删除。
6. 确认后文档从列表消失。
7. 检查 `storage/documents/` 和 `storage/pages/` 中对应文件已删除。

## 验收通过标准

- 后端契约测试通过。
- 前端生产构建通过。
- `uvicorn main:app` 启动方式仍然可用。
- API 路径和响应字段没有因为拆分发生变化。
- 后端不会因为坏 PDF 或 LLM 错误崩溃。
- 前端所有主要操作都有明确状态提示。
- 课程简介和逐页讲稿状态能正确显示。
- 当前页问答按页面隔离。
- 试卷答题页不会提前暴露答案。
- 试卷结果页刷新后仍能恢复判分结果。
- 阶段考试可以从前端创建、刷新、开始和删除。
- 删除文档、试卷或阶段考试不会留下关联错题、答题记录或题目数据。
- 重命名和删除行为符合预期。
