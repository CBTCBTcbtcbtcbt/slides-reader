# 考试功能合并加固计划

本文档记录合作者新增的考试、阶段考试、错题本功能在 merge 前需要完成的加固项。
目的是把剩余工作从对话上下文中移到仓库文档里，避免后续修改只依赖记忆。

## 目标

考试 feature 合并前应满足以下要求：

- 后端数据一致性可靠：删除文档、删除试卷、删除阶段考试时，不留下孤儿试卷、题目、答题记录或错题。
- 异步生成失败可见：普通试卷和阶段考试后台任务失败时，数据库状态必须进入 `failed`，日志里必须保留异常堆栈。
- 前端能管理新增业务对象：用户可以在 UI 中刷新、开始、删除阶段考试，失败或误建的阶段考试不能只能靠手动改数据库清理。
- 答题页和结果页可以刷新恢复：答题页不提前拿答案，结果页不依赖前端内存中的临时结果。
- 代码状态能通过当前项目已有验证：后端 `pytest`、前端 `npm run build`、`git diff --check`。

## 已完成加固项

- 后端已补阶段考试删除的级联清理：`phase_exams` 关联的普通 `exam`、题目、答题记录、错题会一起删除。
- 阶段考试生成失败时，已在创建普通 `exam` 后立即写回 `phase_exams.exam_id`，避免 failed exam 变成孤儿记录。
- 普通试卷和阶段考试后台线程包装层已补 `logging.exception` 和 `failed` 状态兜底。
- 答题页读取题目时可以隐藏 `answer` 和 `explanation`。
- 结果页可以通过 `attempt_id` 重新读取判分结果。
- LLM 设置页已支持保存 `exam_generation_prompt`。

## 本轮完成加固项

### 1. 前端补阶段考试删除入口

已解决问题：
后端已有 `DELETE /api/phase-exams/{phase_exam_id}`，但前端文件页只支持刷新和开始考试。
如果阶段考试生成失败、误创建或不再需要，用户无法从 UI 清理。

完成内容：
- 在 `frontend/src/api/phaseExams.ts` 增加 `deletePhaseExam`。
- 在 `frontend/src/components/FilesView/FilesView.tsx` 的阶段考试列表中增加“删除”按钮。
- 在 `frontend/src/App.tsx` 增加 `handleDeletePhaseExam`，删除成功后刷新阶段考试列表和错题本。

验收：
- TypeScript build 通过。
- 后端已有阶段考试级联删除测试通过。

### 2. 清理阶段考试创建页交互和状态

已解决问题：
`App.tsx` 里 `phaseExamCreating` 目前没有传给任何组件，是死状态。
`PhaseExamCreateView` 在后端只返回 `processing` 时显示“阶段考试生成成功”，文案会误导用户以为试卷已完成。
checkbox 选择逻辑依赖 `li onClick` 加 checkbox `stopPropagation`，虽然能工作，但可读性较差。

完成内容：
- 删除未使用的 `phaseExamCreating` 状态。
- 把创建页成功文案改成“阶段考试已开始生成”。
- 让 checkbox 和整行点击都调用同一个显式处理函数，避免读代码时误以为会双触发。

验收：
- TypeScript build 通过。
- 页面仍能选择多个文档并提交创建请求。

### 3. 修正普通试卷生成 action 状态语义

已解决问题：
`handleGenerateExam` 使用 `DocumentActionState.action = "regeneratingSummary"`，这会把“生成试卷”伪装成“重新生成简介”的忙碌状态。
短期不会导致数据错误，但会降低代码可读性，后续 UI 状态扩展容易误判。

完成内容：
- 在 `frontend/src/types/ui.ts` 的 `DocumentActionState.action` 中新增 `"generatingExam"`。
- `handleGenerateExam` 使用 `"generatingExam"`。
- `FilesView` 中试卷生成按钮根据 `"generatingExam"` 展示“提交中...”。

验收：
- TypeScript build 通过。
- 生成试卷时只有试卷相关按钮显示提交状态，不再复用简介状态。

### 4. 补齐生成任务刷新机制

已解决问题：
普通试卷和阶段考试生成都是异步任务。当前普通试卷生成后会立即刷新一次列表，但任务完成仍依赖用户后续展开/刷新页面。
阶段考试文件页有手动“刷新列表”，但没有自动刷新。

完成内容：
- 阶段考试列表中如果存在 `pending` 或 `processing` 状态，文件页可见时定时调用 `loadPhaseExams`。
- 普通试卷列表如果存在 `pending` 或 `processing` 状态，文件页可见时定时刷新对应文档的试卷列表。
- 复用现有 React `useEffect`，不新增依赖。

验收：
- TypeScript build 通过。
- 轮询只在文件页可见时运行，避免答题页或阅读器页持续刷新无关列表。
